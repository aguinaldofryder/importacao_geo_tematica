## User Story
> **Como** operador, **quero** que o sistema auto-mapeie o máximo possível da planilha contra o catálogo, **para** eu editar manualmente apenas o restante.

## Acceptance Criteria

| # | Critério |
|---|---|
| AC1 | `AutoMapeador.mapear(Classificacao c, ExcelSessao sessao, Fluxo fluxo)` retorna `Mapeamento` |
| AC2 | Para cada coluna dinâmica: match case/trim-configurável entre header e `campo.descricao` (filtrado por fluxo e `ativo='S'`) |
| AC3 | Match único → `status=MAPEADO`, `idcampo` preenchido, `tipo` preenchido |
| AC4 | Match múltiplo → `status=PENDENTE`, `sugestoes=[ids]`, `motivo` explicando |
| AC5 | Nenhum match → `status=PENDENTE`, `motivo` explicando |
| AC6 | Para campos com `tipo=MULTIPLA_ESCOLHA`: lê valores DISTINCT da planilha (via segundo passe ou acumulador) e tenta match contra `alternativa.descricao`. Cada valor sem match → entra em `alternativas` com valor `null` (PENDENTE por alternativa) |
| AC7 | Cobertura de testes unitários ≥ 80% neste módulo (módulo crítico NFR-09) |
| AC8 | Testes unit cobrindo: match único, múltiplos, nenhum, MULTIPLA_ESCOLHA com todas alt ok, MULTIPLA_ESCOLHA parcial, normalização (case/trim), campo inativo ignorado |
