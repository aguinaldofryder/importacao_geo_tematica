## User Story
> **Como** operador, **quero** fornecer configuração em `./config/application.properties` externo ao binário, **para que** eu possa mudar ambiente (dev/hom/prod) sem recompilar.

## Acceptance Criteria

| # | Critério |
|---|---|
| AC1 | `ImportacaoConfig` interface com `@ConfigMapping(prefix="importacao")` mapeia: `saida.diretorio` (default `./saida`), `saida.sufixoTimestamp` (default `true`), `mapeamento.caseSensitive` (default `false`), `mapeamento.trimEspacos` (default `true`) |
| AC2 | Quarkus lê `./config/application.properties` automaticamente no startup (sem flag extra) |
| AC3 | `application.properties.exemplo` criado na raiz com **todas** as chaves comentadas em português (usado como template para FR-20) |
| AC4 | Chaves do datasource seguem padrão Quarkus: `quarkus.datasource.db-kind=postgresql`, `quarkus.datasource.jdbc.url`, `quarkus.datasource.username`, `quarkus.datasource.password` |
| AC5 | Teste unit valida leitura de properties, override e defaults |
| AC6 | **NFR-07:** logs de startup **não contêm** `password` em texto — teste verifica grep no log capturado |
| AC7 | Entrada `config/` adicionada ao `.gitignore` (contém credenciais reais) |
