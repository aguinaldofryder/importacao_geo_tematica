## User Story
> **Como** operador, **quero** rodar `importacao-geo mapear --arquivo X.xlsx --fluxo {territorial|predial}`, **para** gerar `mapping.json` inicial com o que o sistema conseguiu casar automaticamente.

## Acceptance Criteria

| # | Critério |
|---|---|
| AC1 | Comando aceita flags: `--arquivo` (obrigatório), `--fluxo` (obrigatório, territorial ou predial), `--saida` (default `./mapping.json`) |
| AC2 | Fluxo: valida conexão → abre planilha → classifica → auto-mapeia → persiste JSON |
| AC3 | Relatório final ao stdout em PT: `X coluna(s) dinâmica(s) mapeada(s), Y pendente(s). Arquivo: ./mapping.json` |
| AC4 | Relatório lista os PENDENTE com motivo (útil para edição manual) |
| AC5 | Exit 0 mesmo com pendentes (comportamento esperado); exit 1 apenas em erros de infra (conexão, arquivo, etc.) |
| AC6 | Mensagens em PT (NFR-01) |
| AC7 | Teste integração end-to-end em JVM com Testcontainers + amostra .xlsx |
