## User Story
> **Como** sistema, **quero** gerar `UPDATE`s para as colunas fixas, **para** atualizar a tabela principal **sem nunca** emitir `INSERT`.

## Acceptance Criteria

| # | Critério |
|---|---|
| AC1 | `SqlGeradorUpdate.gerar(LinhaMapeada linha, Mapeamento m, Fluxo fluxo)` retorna `Optional<String>` com o SQL (ou vazio se não há colunas fixas a atualizar) |
| AC2 | Output: `UPDATE <tabela> SET col1=<literal1>, col2=<literal2> WHERE <colunaChave> = <literalChave>;` |
| AC3 | Tabela derivada de `fluxo.tabelaPrincipal()` |
| AC4 | **Invariante CON-03:** nenhum caminho de código emite `INSERT INTO tribcadastroimobiliario` ou `INSERT INTO tribimobiliariosegmento` — **teste explícito** faz grep no SQL gerado |
| AC5 | Se nenhuma coluna fixa estiver presente na linha (ou todas forem NULL), retorna `Optional.empty()` — não gera UPDATE |
| AC6 | Se a coerção de qualquer coluna falhar, **não gera UPDATE** e reporta ao chamador (log de erro) |
| AC7 | Testes unit extensivos: happy, todas colunas NULL, coerção falhou em 1 coluna, escape de aspas no valor-chave |
