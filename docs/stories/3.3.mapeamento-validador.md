## User Story
> **Como** sistema, **quero** recusar os comandos `validar` e `importar` se houver qualquer item `PENDENTE`, **para** impedir geração de SQL incompleto.

## Acceptance Criteria

| # | Critério |
|---|---|
| AC1 | `MapeamentoValidador.validar(Mapeamento m)` retorna `ResultadoValidacao(boolean valido, List<String> pendencias)` |
| AC2 | Qualquer coluna dinâmica com `status=PENDENTE` → inválido + pendência descritiva |
| AC3 | Qualquer `alternativa` de MULTIPLA_ESCOLHA com valor `null` → inválido + pendência |
| AC4 | Coluna fixa sem target definido (valor null/vazio) → inválido |
| AC5 | Mensagens de pendência em português, formato `"Coluna X: motivo Y"` |
| AC6 | Método puro — sem IO, sem banco |
| AC7 | Testes unit extensivos cobrindo cada tipo de pendência (módulo crítico NFR-09) |
