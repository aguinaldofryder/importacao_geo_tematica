## User Story
> **Como** sistema, **quero** gerar `INSERT ... ON CONFLICT` para as respostas dinâmicas, **para** fazer upsert idempotente na tabela de respostas correspondente.

## Acceptance Criteria

| # | Critério |
|---|---|
| AC1 | `SqlGeradorUpsert.gerar(LinhaMapeada linha, Mapeamento m, Fluxo fluxo)` retorna `List<String>` (um SQL por campo dinâmico) |
| AC2 | Output por campo: `INSERT INTO <tab_resp> (referencia, idcampo, valor, idalternativa) VALUES (<ref>, <idcampo>, <valor>, <idalt_ou_null>) ON CONFLICT (referencia, idcampo) DO UPDATE SET valor = EXCLUDED.valor, idalternativa = EXCLUDED.idalternativa;` |
| AC3 | Tabela: `fluxo.tabelaRespostas()` (TERRITORIAL→respostaterreno, SEGMENTO→respostasegmento) |
| AC4 | `idalternativa` preenchido **somente** para campos MULTIPLA_ESCOLHA; demais tipos usam `NULL` |
| AC5 | **Idempotência (NFR-06):** teste aplica o mesmo SQL duas vezes contra Testcontainers e confirma que o estado final é igual ao de uma única aplicação |
| AC6 | Campos com valor NULL/vazio podem ser emitidos (atualizando para NULL) **ou** pulados — decisão: pular por default; configurável no futuro |
| AC7 | Testes unit por tipo (TEXTO, DECIMAL, DATA, MULTIPLA_ESCOLHA) + teste idempotência |

## Dev Notes

- `EXCLUDED.*` é a sintaxe PostgreSQL para acessar valores propostos no ON CONFLICT.
- A composição `(referencia, idcampo)` deve ser UNIQUE/PK na tabela de respostas (premissa P3).
