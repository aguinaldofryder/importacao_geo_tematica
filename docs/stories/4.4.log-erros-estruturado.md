## User Story
> **Como** operador, **quero** um arquivo `.log` estruturado com todos os erros por linha, **para** auditar o que falhou e por quê.

## Acceptance Criteria

| # | Critério |
|---|---|
| AC1 | `LogErros.registrar(int numLinha, TipoErro tipo, String detalhe, String contexto)` anexa linha ao `.log` atual |
| AC2 | Cada linha tem timestamp **ISO-8601 com timezone** (NFR-08) |
| AC3 | Enum `TipoErro { MATRICULA_AUSENTE, COERCAO_FALHOU, VALOR_SEM_ALTERNATIVA, ERRO_INESPERADO }` |
| AC4 | Formato da linha: `<ISO-8601> [<TIPO_ERRO>] linha=<n> | <detalhe> | ctx=<contexto>` — humano + parseável (NFR-08) |
| AC5 | Append-only; nunca sobrescreve |
| AC6 | Caminho do arquivo derivado de `ImportacaoConfig.saida.diretorio` + padrão de nome com timestamp |
| AC7 | Thread-safe não é obrigatório em v1 (single-thread) — documentado |
| AC8 | Testes unit: cada tipo de erro, formato ISO, append |
