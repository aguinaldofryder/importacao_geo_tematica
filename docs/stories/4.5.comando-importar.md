## User Story
> **Como** operador, **quero** rodar `importacao-geo importar --arquivo X.xlsx --fluxo territorial --mapeamento mapping.json`, **para** gerar `.sql` e `.log` aplicáveis contra o banco.

## Acceptance Criteria

| # | Critério |
|---|---|
| AC1 | Comando aceita `--arquivo`, `--fluxo`, `--mapeamento` (default `./mapping.json`) |
| AC2 | Fluxo: valida conexão → valida mapping (gate PENDENTE bloqueia) → itera linhas → gera SQL + log → grava arquivos |
| AC3 | Para cada linha: verifica existência da matrícula na tabela principal (SELECT). Ausente → `LogErros.MATRICULA_AUSENTE` + pula toda a linha (FR-10) |
| AC4 | Para linhas válidas: gera `UPDATE` (via 4.2) + `UPSERT`s (via 4.3) e apenda ao `.sql` |
| AC5 | Arquivos de saída: `saida-{fluxo}-{yyyyMMdd-HHmmss}.sql` e `.log` no diretório configurado |
| AC6 | Exit codes: 0 = zero erros, 2 = completou com erros por linha, 1 = falha de infra (conexão, mapping inválido, arquivo ausente) — conforme FR-16 |
| AC7 | Teste integração E2E em JVM: Testcontainers + amostra .xlsx + catálogo seed → confere conteúdo do `.sql` e do `.log` |
| AC8 | **Invariante CON-03 testada no SQL resultante** (grep não encontra `INSERT INTO tribcadastro*`) |

## Dev Notes

- Streaming: não acumular todas as linhas em memória — `try (Stream<Row> s = leitor.linhas())`.
- Gravar `.sql` em buffered writer; `flush` ao final.
- Erros por linha **não interrompem** execução — apenas são registrados.
- Falhas de infra **interrompem** imediatamente.
- Primeira linha do `.sql`: comentário com metadados (arquivo origem, fluxo, timestamp).
