## Dev Notes

### Decisões arquiteturais

1. **Contadores `int` primitivos, single-thread:** PRD NFR-09 exige rastreabilidade, não concorrência. `AtomicInteger` seria defensivo mas desnecessário para a CLI single-thread atual. Documentar em Javadoc para não surpreender futuros devs. Decisão: `int` primitivo, sem lock.

2. **`ResumoSnapshot` com `@RegisterForReflection`:** Record Java serializado por Jackson para a linha JSON do `.log` (Story 5.2) requer entrada em `reflect-config.json` para GraalVM native. Story 5.1 ainda não serializa, mas o record deve estar preparado. Decisão: anotar preventivamente; verificar `reflect-config.json` no T2.

3. **Limitação v1 — `respostasAtualizadas` = 0:** `SqlGeradorUpsert` gera pares DELETE+INSERT; a distinção "par já existia" vs "par novo" só é conhecível em tempo de execução do SQL (fora do escopo de um gerador offline). Para v1, `registrarRespostaInserida()` é chamado N vezes (N = pares gerados) e `respostasAtualizadas` permanece 0. O método existe para compatibilidade com a arquitetura §7 e futuras extensões. Documentar em Javadoc de `ResumoExecucao` e no JSON output de Story 5.2.

4. **`toResumoImutavel(Path, Path)` recebe paths como parâmetro:** Os caminhos de artefatos são conhecidos após a escrita (Fase 5). Passar como argumento mantém `ResumoExecucao` focado em contadores, sem acoplamento às decisões de I/O do `ImportarCommand`.

5. **`IllegalStateException` em `duracao()` e `toResumoImutavel()` antes de `finalizar()`:** Falha explícita > retornar `Duration.ZERO` silenciosamente. Facilita diagnóstico em testes e garante que `ImportarCommand` chama `finalizar()` na ordem correta.

6. **`ResumoExecucao` como variável local (não-CDI):** Stateful por execução — uma instância por invocação de `importar`. CDI `@Dependent` introduziria complexidade sem benefício. Padrão consistente com `LogErros` (Story 4.4) e `boolean[] houveErros`.

7. **Posicionamento de `iniciar()` antes do `try (ExcelSessao)`:** `duracao` em §7.1 representa o tempo da importação (iteração + escrita dos artefatos), não o tempo total do processo CLI (fases de validação). Decisão: `iniciar()` imediatamente antes do bloco de iteração.

8. **`resumo` passado como parâmetro de `processarLinha()`:** Referência de objeto é effectively-final — mesmo padrão do `logErros` e `sqlBuffer` já usados. Sem necessidade de campo de instância.

## Trade-offs registrados

- **`int` vs `AtomicInteger`:** clareza e aderência ao modelo single-thread > defensividade futura; decisão documentada em Javadoc.
- **`IllegalStateException` vs `Optional<Duration>`:** falha explícita > silêncio; facilita diagnóstico sem complicar a API pública.
- **`toResumoImutavel(Path, Path)` vs setter separado:** argumento no momento da criação do snapshot > setter mutável; garante que paths nunca sejam `null` no record.
- **`ResumoSnapshot` record vs classe imutável manual:** record Java 16+ fornece `equals`, `hashCode`, `toString` gratuitamente; adequado para GraalVM com `@RegisterForReflection`.

## Testing

- **Framework:** JUnit 5 + Quarkus test runner (modo JVM)
- **Localização:** `src/test/java/br/com/arxcode/tematica/geo/geracao/ResumoExecucaoTest.java`
- **Tipo:** unitário puro — sem CDI, sem banco, sem I/O de arquivo
- **Cenários obrigatórios:**
  1. Estado inicial: todos os contadores = 0, `inicio` e `fim` nulos
  2. `incrementarLido()` × 3 → `lidos == 3`; demais contadores = 0
  3. `registrarSucesso()` incrementa apenas `sucesso`
  4. `registrarErro()` incrementa apenas `erro`
  5. `registrarUpdatePrincipal()` incrementa apenas `principalAtualizados`
  6. `registrarRespostaAtualizada()` incrementa apenas `respostasAtualizadas`
  7. `registrarRespostaInserida()` incrementa apenas `respostasInseridas`
  8. `totalRespostas()` == `respostasAtualizadas + respostasInseridas`
  9. Acumulação mista: 5 lidos, 3 sucesso, 2 erro, 3 updates, 0 atualizadas, 10 inseridas → snapshot reflete todos os campos corretamente
  10. `duracao()` retorna `Duration` positivo após `iniciar()` + `Thread.sleep(10)` + `finalizar()`
  11. `duracao()` lança `IllegalStateException` se `finalizar()` não foi chamado
  12. `toResumoImutavel()` lança `IllegalStateException` se `finalizar()` não foi chamado
  13. `toResumoImutavel(path1, path2)` — todos os campos do snapshot correspondem ao estado do acumulador
  14. `ResumoSnapshot.totalRespostas()` == soma dos dois campos
  15. `ResumoSnapshot.duracao()` == `Duration.between(inicio, fim)` do snapshot
