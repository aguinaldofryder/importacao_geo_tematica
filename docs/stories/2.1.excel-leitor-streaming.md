## User Story
> **Como** sistema, **quero** ler planilhas `.xlsx` em streaming, **para** processar arquivos com ≥ 100k linhas sem esgotar memória.

## Acceptance Criteria

| # | Critério |
|---|---|
| AC1 | `ExcelLeitor.abrir(Path xlsx)` retorna `ExcelSessao` (AutoCloseable) |
| AC2 | `ExcelSessao.cabecalhos()` retorna `List<String>` (primeira linha) |
| AC3 | `ExcelSessao.linhas()` retorna `Stream<Map<String,String>>` onde chave é o cabeçalho e valor é a célula como string bruta |
| AC4 | Iteração é **streaming** — teste com planilha sintética de 100k linhas não excede 512 MB (NFR-04) |
| AC5 | Arquivo inexistente → `ImportacaoException` com mensagem PT |
| AC6 | Arquivo que não é `.xlsx` → `ImportacaoException` com mensagem PT |
| AC7 | Planilha vazia (só cabeçalho) → `linhas()` retorna Stream vazio |
| AC8 | Testes unitários com planilhas sintéticas pequenas + teste de perf com planilha grande gerada em memória |

## Dev Notes

- `fastexcel-reader`: `ReadableWorkbook.getActiveSheet().openStream()` retorna `Stream<Row>`.
- Coerção de valores: todas as células retornadas como `String` bruta — coerção de tipo é responsabilidade da Story 4.1.
- Primeira linha sempre tratada como cabeçalho (confirmar com usuário se há variantes).
