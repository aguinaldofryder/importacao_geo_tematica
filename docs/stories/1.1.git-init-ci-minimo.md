## 6. Dev Notes

### 6.1. Por que @devops lidera essa story

Operações de push, criação de repositório GitHub e configuração de branch protection são **exclusivas** do @devops conforme `.claude/rules/agent-authority.md`. @dev pode editar `.gitignore` e `ci.yml` localmente, mas apenas @devops executa `git push` e `gh repo create`.

### 6.2. Ordem de execução crítica

```
1. estender .gitignore
2. git init + git branch -m main
3. REVISÃO MANUAL do git status ← não pular
4. git add seletivo
5. commit inicial
6. gh repo create + push
7. criar workflow ci.yml + commit + push
8. atualizar README + commit + push
9. branch protection (se aplicável)
```

**Se o passo 3 falhar** (arquivo sensível aparece como tracked), **não prosseguir**. Ajustar `.gitignore`, usar `git rm --cached` se necessário, e revalidar.

### 6.3. Arquivos sensíveis conhecidos neste workspace

| Arquivo | Situação | Ação |
|---|---|---|
| `.env` | `-rw-------`, contém credenciais | **Nunca commitar.** Já coberto pelo `.gitignore` atual |
| `.env.example` | Pode ter placeholders | **Revisar conteúdo** antes de commitar — confirmar que não há segredos reais |
| `config/application.properties` | Ainda não existe | Garantir que `.gitignore` já cobre antes de qualquer criação |
| `.~lock.TABELA_*.xlsx#` | Lock files do LibreOffice | Adicionar ao `.gitignore`: `.~lock.*` |

### 6.4. Decisões deixadas para Story 1.2

- Versão específica de Quarkus e Java
- Inclusão dos wrappers Maven (`mvnw`, `mvnw.cmd`, `.mvn/wrapper/`)
- Estrutura de pacotes Java e convenções de código formais
- Upgrade do CI para rodar `./mvnw test` real

### 6.5. Decisões deixadas para Story 1.3+
