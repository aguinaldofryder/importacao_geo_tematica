## User Story
> **Como** operador, **quero** ver o resumo formatado no terminal **e** ter a mesma informação em JSON no `.log`, **para** consumo humano rápido e por scripts/parsers.

## Acceptance Criteria

| # | Critério |
|---|---|
| AC1 | `ResumoRenderer.renderizarAscii(ResumoSnapshot r)` retorna string ASCII conforme Arquitetura §7.1 (moldura `====`, alinhamento de colunas) |
| AC2 | `ResumoRenderer.renderizarJsonLine(ResumoSnapshot r)` retorna linha JSON compacta conforme Arquitetura §7.2 |
| AC3 | `ImportarCommand` imprime ASCII no stdout ao final e anexa (texto + linha JSON) ao `.log` |
| AC4 | Exit codes implementados conforme FR-16: `0` sem erros, `2` com erros por linha, `1` falha de infra |
| AC5 | Testes unit: fixture-match para ASCII (linha-a-linha) e JSON (chaves + valores) |
| AC6 | Teste E2E verifica exit code = 0 no happy path e = 2 com linhas erradas |

## Dev Notes

- Arquitetura.md §7.1 tem o modelo exato (moldura, largura de colunas).
- JSON compacto sem pretty-print — uma linha, fácil de grep.
- Exit code no picocli: retornar `int` do `call()`.

## Testing
- Fixture-match é frágil se o texto exato for ajustado — considerar asserts por conteúdo (contains) em vez de equals.
