## User Story
> **Como** equipe, **quero** CI que compile o binário nativo em Linux **e** Windows em paralelo, **para** entregar dois binários a cada release.

## Acceptance Criteria

| # | Critério |
|---|---|
| AC1 | Workflow `.github/workflows/native-build.yml` dispara em push para `main` e em tag `v*.*.*` |
| AC2 | Matrix strategy: `ubuntu-latest` + `windows-latest`, cada um roda `./mvnw package -Pnative` |
| AC3 | Cache de `~/.m2` e da instalação GraalVM |
| AC4 | Artefatos nomeados `importacao-geo-linux-x64` e `importacao-geo-windows-x64.exe` via `actions/upload-artifact` |
| AC5 | Job falha se binário > 100 MB (NFR-05) — checagem explícita no workflow |
| AC6 | Tempo por build < 15 min (orçamentado no plano) |
| AC7 | Smoke test dentro do CI: executa `binario --help` e confirma exit 0 |
